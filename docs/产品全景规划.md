# 梦之书 DreamBook - 产品全景规划

> 记录你的梦，读懂你的潜意识。
> 胃之书记录你吃了什么，梦之书记录你梦了什么。

## 产品定位

一款以 AI 驱动的梦境记录与解析 iOS App。用户记录梦境，AI 提供多流派深度解析与跨梦境洞察，叠加多模态内容生成（图片、视频），打造"越用越懂你"的个人潜意识智能系统。

### 核心价值主张

- **对用户**：不只是记梦工具，是你的私人潜意识分析师。累积的梦境记录让 AI 越来越懂你，每一条新梦境都让过去的记录更有价值
- **对市场**：解梦需求已充分验证，但现有产品"用完即走"、审美粗糙。用设计品味和 AI 累积洞察碾压存量市场
- **对作者**：AI 产品经理的作品集项目，展示 AI 多模态产品的全链路设计与落地能力

### 设计北极星

参考「胃之书」的产品哲学：把平凡的个人数据（美食/梦境），通过精致的视觉化和 AI 洞察，变成有情感价值的个人资产。

### 核心壁垒

单次解梦没有壁垒，累积的个人上下文才是壁垒。每多记一个梦，AI 对你的理解就更深一层，洞察就更精准——这是"用完即走"型竞品永远做不到的。

## 技术栈

| 层级 | 方案 | 备注 |
|------|------|------|
| 客户端 | SwiftUI（原生 iOS） | — |
| 后端服务 | **待调研** | 要求：新手友好、国内网络友好。候选：LeanCloud / 腾讯云开发 / 阿里云Serverless 等，需进一步调研对比 |

| 对象存储 | 随后端方案确定 | 存储生成的图片/视频 |

## 功能全景图


### 一、记录梦境 — 输入层

> 一切的起点。核心设计原则：刚醒来、意识模糊时也能 30 秒内完成记录。

| 功能 | 描述 |
|------|------|
| 语音输入 | 主要输入方式。醒来后直接说，Apple Speech 端侧识别，无需联网 |
| 文本输入 | 辅助输入。支持清醒后对语音识别结果进行编辑修正 |
| 原始语音保留 | 保存录音文件，方便用户回听原始描述并修正文字 |
| 梦境元数据 | 日期/时间自动记录；用户可补充：情绪标签、清晰度评分、是否清醒梦 |
| AI 关键词提取 | 自动从梦境文本中识别人物、地点、物品、情绪等关键要素 |
| 个人词典 | 反复出现的人/地/物自动积累为个人词典，提升后续识别准确度，服务于统计和解梦上下文 |


### 二、梦境画廊 — 展示层

> 让记录变得有价值感。设计目标：每次打开 App 都觉得"这些梦值得被记住"。

| 功能 | 描述 |
|------|------|
| 梦境卡片列表 | 首页核心。每张卡片呈现：文本摘要、情绪色彩、日期、AI 配图缩略图。参考胃之书的卡片视觉化处理 |
| 梦境详情页 | 完整梦境文本 + 解析内容 + 多模态内容 + 更多解析入口 + 更多生成入口 |
| 时间轴浏览 | 按日/周/月浏览梦境，追求沉浸的时光回溯感 |
| 搜索与筛选 | 按关键词、情绪、日期范围检索梦境 |
| 日历视图 | 日历上标记记梦日，形成"打卡"视觉反馈 |


### 三、AI 解梦 — 核心价值层

> aha moment 的制造机。这是产品的灵魂。

| 功能 | 描述 |
|------|------|
| 单梦解析 | 对单条梦境进行深度分析，挖掘象征意义和潜意识线索 |
| 多流派解梦师 | 周公（中式古典）、弗洛伊德（精神分析）、荣格（原型理论）及各文化特色解梦师 |
| 解梦师专属 UI | 每位解梦师有独立的视觉呈现：周公→竹简/卦词，弗洛伊德→手写信/羊皮纸，荣格→原型图谱等 |
| 跨梦境周报 | 结合一周内所有梦境，发现主题关联和情绪趋势 |
| 跨梦境月报 | 月度全景分析，更深层的潜意识模式识别 |
| 个人上下文累积 | 解梦时自动引用历史梦境中的关联元素，越用越懂你 |
| 梦境符号词典 | 个人专属 + 通用符号解释库，随使用自动丰富 |


### 四、多模态生成 — 传播层

> 让梦境可见、可分享。传播的燃料。

| 功能 | 描述 |
|------|------|
| 梦境配图生成 | 基于梦境内容生成梦核（dreamcore）风格图片 |
| 梦境视频生成 | 短视频，梦幻感、粒子碎片重组的视觉风格 |
| 分享卡片生成 | 精美图文卡片，一键分享到社交平台 |
| 风格选择 | 可选视觉风格：超现实主义、水彩、赛博朋克、水墨等 |


### 五、统计与洞察 — 数据层

> 累积价值的可视化。让用户看到"我的梦境资产"在增长。

| 功能 | 描述 |
|------|------|
| 记梦频率统计 | 本周/本月记梦数量、连续记录天数 |
| 情绪趋势图 | 梦境情绪随时间的变化曲线 |
| 高频主题/符号 | 词云或排行榜，最常出现的梦境元素 |
| 梦境图谱 | 关键词之间的关联网络可视化 |
| 年度梦境报告 | 年终总结式梦境回顾，传播向 |


### 六、伪社区 — 冷启动层

| 功能 | 描述 |

### 七、商业化 — 变现层

> 积分制，免费保留存，付费解锁 AI 深度能力。

| 功能 | 描述 |

| 付费能力 | 解梦消耗积分、多模态生成消耗积分、高级统计/周月报解锁 |
| 积分包购买 | 不同档位积分包，通过 iOS IAP（应用内购买）完成 |


### 八、账户与基础设施

## 留存飞轮

> 这个品类的原罪是"用完即走"。飞轮设计是生死线。

```
晨间提醒"记录昨晚的梦"
    ↓
语音快速记录（30 秒内完成）
    ↓
AI 即时返回一段简短解析（免费钩子，激活好奇心）
    ↓
想看更深入的多流派解析？→ 消耗积分（付费转化点）
    ↓
周末推送"本周梦境周报"（展示累积价值）
    ↓
月底推送"月度潜意识画像"（aha moment 制造）
    ↓
年底"年度梦境报告"（传播事件 + 情感锚点）
```

**飞轮核心逻辑**：每一条新梦境都让之前所有梦境更有价值。用户的梦境资产在增长，离开成本也在增长。

## 品牌与设计语言

### 品牌关键词

梦之书 / 捕梦网

### 视觉原则

- **暗色为主**：适配记梦场景（刚醒来、灯光昏暗），降低视觉刺激
- **梦幻感**：粒子、光晕、渐变、半透明层叠，传达梦境的朦胧与流动
- **神秘感**：解梦模块需要仪式感，每个文化流派都有独特的视觉语言
- **信息卡片视觉化**：参考胃之书，把文本信息转化为有情感的视觉卡片

### 交互原则

- **极简记录流**：语音按钮一触即录，梦醒时刻不需要任何认知负担
- **生成时的仪式感**：AI 解析和多模态生成时，用粒子碎片重组的动效传达"梦境正在被解读"
- **底部导航**：Tab 与新建按钮平级（参考胃之书），记录入口永远触手可及

## 待调研项

- [ ] **后端服务选型**：需要对比 LeanCloud / 腾讯云开发 / 阿里云 Serverless 等方案，评估维度：新手友好度、国内网络、免费额度、文档质量、iOS SDK 支持
- [ ] **LLM 具体选型**：DeepSeek / 智谱 / Kimi 等国产模型的 API 定价、响应速度、解梦场景效果对比
- [ ] **图片生成模型对比**：CogView / 通义万相 / 其他方案的出图质量、风格可控性、API 成本
- [ ] **视频生成可行性**：可灵 API 的接入成本、生成时长、质量是否满足预期
- [ ] **App Store 审核要求**：AI 生成内容的审核政策、积分制 IAP 的合规要求
